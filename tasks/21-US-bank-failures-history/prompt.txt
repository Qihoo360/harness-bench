# 美国银行倒闭潮历史图表分析任务

## 1. 任务背景
宏观经济研究团队提供了一张自1934年以来美国各年银行倒闭数量的柱状图（位于 `fixtures/in/` 目录下）。你需要作为金融历史分析师，通过视觉读取图表中的数据分布特征，并结合外部宏观经济史知识，分析图中出现的几次重大“倒闭潮”。

## 2. 核心子任务 (Subtasks)
- 图表时间轴与峰值定位：观察柱状图的分布形态与 X 轴刻度，准确推算出历史上出现的三波明显的“银行倒闭高峰”各自所处的大致年份区间。
- 宏观经济历史背景检索：针对识别出的每一个倒闭高峰时间段，通过知识检索，查明导致该时期大量银行集中破产的底层宏观经济事件或金融危机名称。
- 结构化分析报告生成：将每个阶段的时间段与对应的时代背景整合成符合要求的 JSON 结构数据。

## 3. 产出物定义 (Required Artifacts)
任务完成后，必须在当前工作目录下产出以下文件：
- `result.json`: 包含三波倒闭潮分析的结构化文件。必须包含一个名为 `crises` 的数组，数组内需包含 3 个对象（按时间先后排序），每个对象有且仅有两个键：
  - `time_period`（字符串类型，描述该波高峰的年份区间，例如："19xx-19xx年代"）
  - `background`（字符串类型，该时期爆发倒闭潮的核心历史事件与时代背景简介，不少于 50 字）
- `progress.md`: 记录子任务执行过程的文档，必须包含你推算 X 轴年份刻度的逻辑、知识检索使用的关键词以及初步归纳的草稿。

## 4. 执行约束 (Constraints)
- 零幻觉原则：高峰时间段的推断必须具备图表视觉支撑（注意横坐标是不等距/有省略的，需仔细推算），时代背景必须基于真实的经济史实，严禁捏造。
- 格式规范：`result.json` 必须是合法的 JSON 格式，数组长度必须严格为 3。
