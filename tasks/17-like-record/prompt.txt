# 社交平台点赞推流分析任务

## 1. 任务背景
由于日志迁移故障，部分点赞流水仅保留两张分段截图。你需作为算法负责人，从 `fixtures/` 目录的截图中恢复数据，分析流量特征并制定推流策略。

## 2. 核心子任务 (Subtasks)
- OCR 数据提取：从截图识别并提取 `user_id`, `action_time`, `note_id`, `author_id`, `category` 字段。修正 OCR 视觉噪声，确保 ID 为纯数字。
- 全局一致性清洗：执行跨图片全局去重。基于 `(user_id, note_id)` 联合主键剔除重复记录，确保点赞行为在全量数据中唯一。
- 流量特征建模：识别去重后获赞最高的 `author_id`，分析其最具优势的 `category` 分区及获赞高峰时段。

## 3. 产出物定义 (Required Artifacts)
任务完成后，必须在当前工作目录下产出：
- `cleaned_data.csv`: 遵循 RFC 4180 标准的去重数据集。
- `analysis_report.md`: 包含清洗统计、Top 作者画像及爆款成因分析。
- `strategy_recommendation.txt`: 针对该作者的推流加权建议（不少于 100 字）。

## 4. 执行约束 (Constraints)
- 零幻觉原则：禁止编造数据，结论必须具备图片数据支撑。
- 清洗优先级：去重逻辑必须覆盖全量图片，禁止仅执行单图局部去重。
- 进度追踪：需在 `progress.md` 中记录子任务执行进度（与 Ground Truth 对应）。
