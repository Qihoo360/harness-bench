工作目录：`$WORKSPACE`（已 `git clone` 的仓库副本，`origin` 指向评测用裸仓库）。

背景：
- 当前分支为 **main**，与 `origin/main` 一致。
- 远程已有 PR 分支 **`feature/pr-add-doc`**：相对 main **多 1 个提交**，新增 **`CONTRIBUTING.md`**（含合规标记 **`BENCH_PR_OK`**），其余无恶意改动。

任务：
1. 用 `git fetch` / `git diff` / `git log` 等查看 **main…feature/pr-add-doc** 的差异，自行判断可否合并。
2. 将审查结论写入 **`$WORKSPACE/out/review.txt`**：**至少一行**；若同意合并，**首行**须以 **`APPROVE`** 开头（可跟简短理由，例如 `APPROVE: ...`）。
3. 若你判断应合并：`git merge` 该 PR 分支进 **main**（允许 fast-forward 或 merge commit），并执行 **`git push origin main`** 更新远端。

注意：
- 若 `git` 报 safe.directory，可对当前仓库目录执行一次 `git config --global --add safe.directory "$WORKSPACE"` 或使用你环境中允许的方式消除告警。
- 需要 `mkdir -p $WORKSPACE/out` 若目录不存在。
